## ✅ Выполненные задачи

### 1. Локализация (RU/EN)
- ✓ Создана система локализации на сессиях
- ✓ Загружаемые файлы локали: `locales/ru.json`, `locales/en.json`
- ✓ Сервис `LocalizationService` с поддержкой детектирования страны
- ✓ Endpoints для переключения языка:
  - `GET /api/locale` - получить текущую локаль и переводы
  - `POST /api/locale` - изменить локаль (на сессию)
  - `GET /api/locale/detect` - детектировать локаль по стране
- ✓ UI элемент - всплывающая плашка с выбором языка
- ✓ Кука умирает с закрытием браузера (session-based)

### 2. Система оценок и комментариев
- ✓ Интерфейс Review с полями: id, productId, userId, username, rating (1-5), comment, createdAt
- ✓ Endpoints:
  - `POST /api/reviews` - добавить/обновить отзыв (требует авторизации)
  - `GET /api/reviews/:productId` - получить все отзывы и среднюю оценку
- ✓ Отображение на карточке товара:
  - Средняя оценка
  - Список отзывов с датами
  - Форма для добавления отзыва (только для авторизованных)
- ✓ Одна оценка на пользователя на товар

### 3. Система рекомендаций с тегами
- ✓ Интерфейс UserLikes с полями: userId, likedProductIds, recommendedTags, lastUpdate
- ✓ Endpoints:
  - `POST /api/likes/:productId` - лайкнуть товар (добавить в рекомендованные)
  - `GET /api/recommendations` - получить рекомендованные товары
- ✓ Логика:
  - При лайке товара, его теги добавляются в рекомендованные
  - Рекомендации выстраиваются по похожим тегам
  - Рекомендации "умирают" через 3 дня неактивности
  - Товары не дублируются с теми, что уже liked или owner

### 4. Ролевая система (Admin)
- ✓ Расширено поле role в User: 'user' | 'admin'
- ✓ Middleware `requireAdmin` для защиты admin endpoints
- ✓ Admin endpoints:
  - `POST /api/admin/products` - создать товар (только админ)
  - `PUT /api/admin/products/:id` - обновить товар (только админ)
  - `GET /api/admin/products` - получить все товары администратора
- ✓ Admin интерфейс на фронте:
  - Страница администратора (`#admin`)
  - Форма для создания и редактирования товаров (не in-cell)
  - Список своих товаров с кнопками редактирования/удаления
- ✓ Сессия админа умирает через 1 час неактивности

### 5. Фронт-енд (на ванильном TS)

## 🔑 Основные API Endpoints

### Локализация
- `GET /api/locale` - получить текущую локаль
- `POST /api/locale` - изменить локаль
- `GET /api/locale/detect` - детектировать локаль

### Товары
- `GET /api/products` - список товаров с фильтрацией
- `POST /api/products` - добавить товар
- `GET /api/my-lots` - мои товары
- `DELETE /api/my-lots/:id` - удалить товар

### Отзывы
- `POST /api/reviews` - добавить отзыв
- `GET /api/reviews/:productId` - получить отзывы товара

### Рекомендации
- `POST /api/likes/:productId` - лайкнуть товар
- `GET /api/recommendations` - получить рекомендации

### Корзина
- `GET /api/cart` - получить корзину
- `POST /api/cart` - добавить в корзину
- `PUT /api/cart/:productId` - обновить количество
- `DELETE /api/cart/:productId` - удалить из корзины
- `POST /api/cart/checkout` - оформить заказ

### Покупки
- `GET /api/inventory` - история покупок
- `POST /api/buy/:id` - купить товар напрямую

### Admin (требует role: 'admin')
- `POST /api/admin/products` - создать товар
- `PUT /api/admin/products/:id` - обновить товар
- `GET /api/admin/products` - получить свои товары

## 📝 Как использовать новые функции

### 1. Локализация
- При открытии сайта на первый раз показывается всплывающая плашка с вопросом о локали
- Пользователь может выбрать "Да" или "Нет"
- Если "Нет" - язык переключается на противоположный
- Выбор сохраняется в сессию (умирает с закрытием браузера)

### 2. Оценки и отзывы
- На карточке товара отображается средняя оценка ⭐
- Кликнув на товар можно оставить или обновить отзыв
- Форма отзыва с 5-звездочной шкалой и текстовым комментарием
- Видны все отзывы с именем пользователя и датой

### 3. Рекомендации
- Кнопка "❤️ Like" на карточке товара
- При лайке товара его теги добавляются в рекомендованные
- На главной странице рекомендации встроены в обычный список товаров
- Рекомендации "умирают" через 3 дня

### 4. Admin панель
- После регистрации админа, на боковой панели появляется "Панель администратора"
- На админ странице видны все товары с кнопками редактирования и удаления
- Кнопка "+ Добавить товар" для создания нового товара
- Форма редактирования с полями: имя, цена, количество, персонаж, редкость, описание, теги

## 🔐 Безопасность

- Пароли хранятся в открытом виде (для учебного проекта)
- Сессии защищены httpOnly cookie
- Admin доступ требует проверки role
- CORS не настроена (локальный проект)
- Session middleware защищает от CSRF
